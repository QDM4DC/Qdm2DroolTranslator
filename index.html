# About QDM Phenotyping is a tool suite to automate the transformation of QDM/HQMF XML into Drools Rules, and execute them against a Hadoop data source. ![Architecture](assets/img/htp-drools-arch.png) # Installation ## Build/Compile 1. Install [Git](http://git-scm.com/book/en/Getting-Started-Installing-Git) 2. Install [Maven](http://maven.apache.org/download.cgi) 3. Clone the repository using a Git Clone ```https://github.com/SHARP-HTP/qdm-phenotyping.git``` 4. From the ```qdm-phenotyping``` directory, run ```mvn clean install``` Artifacts will be available in the ```target``` directory of the modules. ## Maven Add the following repository to our ```pom.xml``` file: ```xml <repository> <id>informatics-releases</id> <url>http://informatics.mayo.edu/maven/content/repositories/releases</url> </repository> ``` Then add in the dependency: ```xml <dependency> <groupId>edu.mayo</groupId> <artifactId>qdm-{module}</artifactId> <version>1.0.0.RELEASE</version> </dependency> ``` ___NOTE:___ Replace the ```{module}``` placeholder with the name of one of the [modules](#toc4) listed below. # Modules #### qdm-cypress A module for interacting with data from the [Cypress Project](http://projectcypress.org/). Functionality includes transforming Cypress patient data into the qdm-phenotyping model, and validating population results against expected values. #### qdm-executor QDM Executor is responsible for compiling QDM XML into executable Drools Rules and executing these rules against a patient set. #### qdm-demographics A processing module for aggregating Demographics data from a set of processed Patients. #### qdm-patient-model QDM Patient Model is a Java Bean representation of secondary use patient data. This model is optimized to be used with the ```qdm-executor``` module, and is not intended to be a standard patient data format for transport or otherwise. #### qdm-webapp A web application exposing the QDM Phenotyping API via REST Services. #### qdm-cli QDM CLI is the command-line interface into the QDM Phenotyping project. # API A web application exposing the QDM Phenotyping API via REST Services. ```java /** * Execute a given set of patients in a single session. * * @param patients the patients * @return the results */ public void execute(Iterable<Patient> patients, String qdmXml, MeasurementPeriod measurementPeriod, Map<String,String> valueSetDefinitions, ResultCallback callback); ``` # License All artifacts are licensed under the [Apache License](http://www.apache.org/licenses/LICENSE-2.0.txt).
